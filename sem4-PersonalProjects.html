Secure Solution Project for Rotterdam Port

My group and I worked on a defensive cybersecurity project for Rotterdam Port, aiming to provide safe infrastructure. This effort was motivated by a prior ransomware attack on the port, which highlighted the need for improved cybersecurity measures.

Background: Rotterdam Port Cyber Attack

The Rotterdam Port, along with other entities such as APM container terminal, MSD pharmaceutical, and TNT package carrier, fell victim to a ransomware attack. The attack paralyzed operations, emphasizing the imperative for robust cybersecurity measures to prevent such incidents in the future.

Client Meeting and Requirements

We initiated the project by meeting with our client from the Rotterdam Port. During this meeting, the client outlined specific requirements for the secure solution, informed by the recent cyber attack. The primary goal was to establish a resilient and proactive cybersecurity infrastructure to mitigate future risks effectively.

Secure Solution Overview

Our proposed secure solution incorporates several key elements:

  1. Network Segmentation with 8 VLANs: Each VLAN is equipped with a pfSense firewall and Suricata with advanced Snort rules to detect malicious activities, here is a diagram. üëâ(Client explained how the virus spread so quickly, because his whole infrastructure had no segmentation in their network)
  2. Email Server with Phishtool: An email server for the use of the Phishtool, which is utilized for analyzing and reverse engineering emails to identify malicious content and employee training video on what to do when you see a malicious email.. üëâ(Client wanted more email security, the main culprit of the whole attack was a Phishing email)
  3. Monitoring Infrastructure:
    • Nagios XI: Employed to monitor the entire infrastructure for any anomalies.
    • WaZuh: Added agents to each machine to monitor and take action in the event of suspicious activity. Here is a video of WaZuh taking action against a suspicious file WaZuh demo
    • Zeek: Implemented to monitor the network layer across all devices.
    • Honeypot: Implemented a honeypot in the DMZ section of the network to further monitor any malicious activity.

Implementation Process

After receiving the client's requirements, we began by creating a network diagram and discussing it with the client for validation. The diagram included IP addresses, firewall structures, and chosen monitoring tools. During implementation, we focused on clear machine structures, naming conventions, and configuration based on predetermined IP addresses.

Reflection on the Project

Reflecting on the Secure Solution Project, we encountered both successes and challenges. Simulating the scenario of creating a project for a real client was both enjoyable and instructive. The project facilitated a deeper understanding of working collaboratively in a group, and each team member gained insights into their respective tasks.

Conclusion

The Secure Solution Project was a fun project to do. The implementation process involved precise planning, client collaboration, and effective use of cybersecurity tools. As we conclude this project, we carry forward valuable lessons learned and a heightened capability to address cybersecurity challenges in real-world scenarios. *(this project was a hypothetical project given by the teachers at Fontys, we didn't really get involved with the Rotterdam Port Organization/Company)
